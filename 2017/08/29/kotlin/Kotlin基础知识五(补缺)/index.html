Kotlin基础知识五(补漏)

发表于 2017-08-29   |   分类于 kotlin

可见性修饰符

可见性修饰符可以用在类，对象，接口，构造函数，方法，属性以及他们的get/set中。

  • private 对于属性，方法，构造函数而已，他们只是会在自身类可见，对于类而已，只是在该文件内可见。
  • protected 他的范围是private+子类可见，同时，对于继承的protected成员，没有显示指定可见性时，还是会是protected的。
  • internal 会在相同的模块内随处可见(即一起编译的模块)
  • public 这是默认的修饰符，任何客户端都可以使用。

扩展

通过扩展，我们可以避免重新继承一个类来重写或者添加新的功能，来达到我们的目的。比如我们需要给某一个类增加一个方法，我们就可以通过扩展，而不用继承。扩展可以扩展属性和扩展方法。

泛型

Kotlin是没有类似java的通配符的，但是他有声明处型变和类型投影。
首先是协变和逆协变的概念。协变是指：类似java的泛型中extands关键字，? extands T 这种，是的参数类型有上界，他可以是本身类型或者是子类型作为参数，叫做协变。反之，在java中 ? super T这种有下界的情况是叫做逆协变。
在kotlin没有super或者extands这种写法，而是引入了生产者-消费者的概念(PECS)。那些只能从中读取的对象叫做生产者(返回值)，而那些只能写入的对象为消费者(参数或者变量)。

out和in

out：一般原则，当一个类C的泛型参数类型是T，而且被声明为out的时候，他只能主线在C成员的输出位置中(不能作为参数，而是作为返回值或者其他不会被处理的情况)，这种情况下C可以安全的作为C的超类。这也就是协变。
int:跟out作用相反，他的泛型参数只能作为参数形式或者其他值的形式在类中进行处理而不能是返回。同样的，可以有类似多态的功能，当一个对象的类型是C的时候，他能够安全的转为子类型C。
函数的上下界之前的文章已经有所提及，这里不做重复。
