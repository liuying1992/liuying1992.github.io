Adb命令

发表于 2017-11-02   |   分类于 adb命令

adb shell 命令是Android系统特有的命令，存放在system/bin目录下面

安装：
1 adb install xxx.apk 需要先卸载
2 adb install -r xxx.apk 保留数据安装，等同于升级安装

卸载：
3 adb uninstall xx pn 完全卸载
4 adb uninstall -k pn 保留数据
显示已经连接的设备
5 adb devices

针对特定的设备操作
6 adb -s 模拟器编号 command
比如，当前是有多设备连接的，我们需要安装到特定的某一台设备上，就需要加上-s命令

7 导入文件到模拟器
adb push file 模拟器的path

8 到处文件
adb pull 模拟器文件 本地路径

9 查看模拟器log信息
adb logcat

9 获取管理员权限
adb root

10 进入shell模式
adb shell

进入shell模式之后，可以很方便的使用linux命令去操作自己app目录下面的相关文件，还可以随便的cp出来。比如，我们需要查看db文件。比如运行run-as packagename 则可以进入我们的对应可以debug的包名的app的根目录下。然后可以cp或者是cat文件，导出数据库等

11 查看分辨率
adb shell dumpsys window display
高通机器使用：adb shell vm size

12 获取帮组
adb help

13 启动/关闭adb 进程
adb start-server
adb kill-server

14 查看adb在设备中所允许的进程
adb shell ps | grep adbd
在windows中，使用findstr替代grep

15 获取设备状态
设备状态有三种，正常连接，出现异常，没有连接设备，使用adb get-state

16 列出设备安装的所有应用
adb shell pm list package [command]
comand是可选的，没有的时候则是列出设备所有安装的应用
-s 列出系统应用
-3 第三方应用
-f 列出应用包名和对应的apk名以及存放的位置，可以跟上面两个一起使用或者单独使用
-i 列出应用包名以及他的安装来源
命名最后可以加关键字过滤，过滤的是包名比如需要查找饿了吗，可以是
adb shell pm list package -f -3 -i me

17 列出对应包名的.apk位置
adb shell pm path packagename

18 adb shell pm dump package 可以列出指定应用的dump信息

19 打开一个网页
adb shell am start -a android.intent.action.VIEW -d website

20 启动拨号器
adb shell am start -a android.intent.action.CALL -d tel:phonenumber

21 监控crash和ANR
adb shell am monitor

22 adb shell am fore-stop packagename
结束应用

23 截屏
adb shell screencap -p /sdcard/filename

24 录制
adb shell screenrecord sdcard/filename

25 查看设备参数信息
adb shell getprop
他是以key-value形式显示，如果需要获取某一个key值
adb shell getprop key

26 具体apk的文件查看，比如db，sp等的导出
adb shell $ run-as packname
之后使用cp 命令，比如把Test.db导出到sdcard
cp ./databases/Test.db /sdcard/

27 wifi调试
首先查看调试的机器需要与电脑位于同一个局域网，然后使用命令
adb tcpip port
adb connect ip:port
切换回usb调试
adb usb
