导入和导出

包括组件，变量，函数的导出，
默认组件导出直接使用export default既可，变量的导出可以是单个导出或者是批量导出，也是使用export关键字，函数导出在function之前添加’export’既可。
他们的导入是在import的时候，组件是独立引入，函数和变量使用{}括起来，函数只需要写名字既可。

Props

  • 对于Props，他是一个对象，他是只读的，纯净的，对于所有的React Component来说，他们必须是纯净的function，不可以对Props做出修改。
    props是组件本身的属性，一般用于嵌套内外层的组件中，负责传递信息(通常是由父组件向子组件传递)在最新版本中，是通过参数添加的形式写入，props对象中的属性和组件的属性是一一对应的。

  • 使用

  • props的合法验证：我们可以给props设置默认值，可以设置props的属性的类型，这样子对于外部输入进行合法性校验。
    目前新版本的react(16以后)已经去除了PropTypes，独立出来另外一个js文件，所以，我们进行props的类型检测需要另外npm(prop-types)

  • 一些操作属性技巧
    延展操作符...：在传入给子组件props的时候，假如是多个，可以组成一个对象，通过...传入

  • 解构赋值操作符：可以只是取对象中的某一部分属性，然后进行读取赋值给别人

State

对于Props，我们一般是用来设置一些不变参数，对于一些DOM变化需要反应到外部的，我们通过State实现，这样子我们才能更好的封装组件。
State和Props很类似，但是State是私有的，他完全由组件控制，不能被外部控制。
State的初始化，可以是在构造器中(建议是)，可以在外部，直接编写。关于state的使用，我们上面有提到。state他类似与一个状态机，当他的状态发生了改变的时候，就会去render()。

ref

ref是用来会去真实DOM节点的，有些时候我们需要直接操作DOM节点才可以，比如说给某一个input获取焦点。
ref的使用有两种

类

对于类，跟Java是十分相似的。他使用constructor标识构造器而非文件名称，然后内容属性可以不用声明，直接使用this进行设置便会生成。被继承的时候，可以对方法进行重写。
