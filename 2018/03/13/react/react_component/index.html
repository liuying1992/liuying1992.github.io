ReactNative Component 基本使用

包括Button，Text，TextInput,View，Img，Touchable系列组件，他们都是位于react-native包下的

Button

Button组件是支持Android和ios平台的。官网地址
他其实是对Text的一层封装，平时应该比较少用，不灵活，直接使用Text组件即可，他的源码是:Button，他的一些样式基本上是写死的。
基本使用：

属性解析：

  • onPress:按钮按下时候的事件响应，参数是一个function
  • title：Button的内容。需要注意的是title不能写在Button标签之间。
  • color:对于iOS 平台是字体颜色，对于Android平台是button 的背景颜色。
  • accessibilityLabel：设置盲人是否可访问，就是方便盲人用的时候的提示，一般用不到。他的值是盲人使用的时候的提示。是辅助功能。
  • disable:是否可用，boolean
  • testId:用来定位在本Component的位置，可以用来查找该Button。
  • 需要注意button没有style属性，假如需要设置，我们需要通过外层包裹Button的View进行设置。

View

View是ReactNative组件当中最基础的组件，它相当于Android的View，iOS的UIView，前端的

一般。可以用来包裹其他组件。它支持使用flex，可以相应事件以及访问性控制。

View 没有onPress事件。但是它有比较多的事件触摸属性以及props，相关可以查看View。

Text

该组件是用来显示文本的。它支持嵌套，样式以及事件处理。官网地址

  • Text可以用来嵌套Text，这时候他是不换行的，他可以起到的作用是类似于Android的SpannableString和iOS的NSAttributedString的作用。但是，当Text嵌套在其他组件之中的时候，是使用flexbox布局的，他会占据一行或者是多行。

  • 所有的文本节点，必须使用Text包裹，而不能直接使用View包裹。同时，我们不能像写html那样子，设置默认字体和颜色等，假如我们有这样子的需求，可以通过自定义Component实现。在RN中，继承样式的概念是可以出现在嵌套Text当中的。

  • 属性解析
    selectable：boolean类选，让用户是否可以使用本机的复制或者粘贴内容。
    accessible：可访问性控制，就是一些盲人提示
    ellipsizeMode：但设置了numberOfLines 属性的时候，他决定了如何截断文本，必须和numberOfLines配合使用，他的值由：head,middle,tail,clip组成

nativeID:查询该View
numberOfLines:设置最多可以显示多少行。当足够位置的时候，Text不会占据满numberOfLines这么多行，而是会wrap内容。
onLayout(x,y,width,height):：当挂载或者是布局发生变化的时候调用，是回调函数。
onPress:点击
‘onLongPress’:长按
disabled:禁用该组件
selectionColor:选择文本hi和的高亮颜色
suppressHighlighting:压制选择图片之后的文字状态，true的时候不会有视觉变化，默认情况下，灰色的椭圆形突出显示的文本。
style:设置Text的样式，他可以定义在StyleSheet中，也可以直接定义在style大括号之内。内容比较多，自己可以常看。

TextInput

输入文本的组件。Props提供了一些基本可配置性的一些属性，比如数字键盘，自动校正，自动大写等。官网地址

Text Input他会有包含一个默认的有背景颜色和padding的底部线，不需要可以通过不设置InputText的高度，这样子就可以通过系统去给bottom line设置正确的位置，或者是设置固定的高度，通过修改underlineColorAndroid为透明transparent。
同时，文本的选择，输入可以改变程序的活动窗口，对于Android来说，Activity的windowSoftInputMode的参数adjustResize将会影响到组件的布局，所以，我们一般需要设置特点的windowSoftInputMode值。

属性：

  • placeholderTextColor 提示hint的字体颜色
  • allowFontScaling 字体是否应该放缩来适应访问性控制。默认是true的
  • autoCorrect 自动校正，默认true
  • autoFocus 自动聚焦，如果是true，将会集中在componentDidMount()方法中，默认是false。
  • defaultValue 默认值
  • blurOnSubmit 对于当行，提交的时候模糊，默认是true，对于多行则为false。注意，对于多行，将模糊提交改为true意味着返回将模糊的字符并且触发onsubmitEditing事件。
  • contextMenuHidden true则隐藏上下文菜单，默认false
  • editable:是否可编辑，默认true
  • keyboardType 弹起键盘的默认类型，比如numeric，这几种是跨平台的值：default，numeric,email-adress,phone-pad。而另外iOS和Android另外有部分。
  • maxHeight 假如autogrow是true的，那么当Text输入的高度到达maxHeight之后，将会以滚动条的形式上下滚动显示。
  • multiline 如果是true，可以输入多行，默认是false
  • onBlur 当Text input模糊的时候被调用
  • onChange当输入发生改变的时候回调
  • onChangeText 当文本发生变化的时候回调，同时会把新的内容文本回调回来。
  • onContentSizeChange 当输入内容的大小发生变化的时候会回调，他将会回调原生的事件：{ contentSize:{width,height}}
  • onEndEditing 输入结束的时候调用
  • onFocus 文本聚焦的时候调用
  • onLayout 挂载或者是布局发生变化的时候调用，回传参数是(x,t,width,height)
  • onScroll 挂载或者滚动时调用contentOffset:{x,y}，但是android不提供contentSize属性。
  • onSelectionChange 选择文本选择改变的时候回调。
  • clear() 清除内容
  • isFocused是否聚焦
    还有其他的一些属性不一一列举，自己慢慢尝试。

Image

用来显示不同类型的图像，包括网络图片，本地图片，原生图片，手机本地图片
对于Android平台，还支持GIF和WebP图片，但是需要添加以下依赖到对应的gradle中

对于GIF图，还需要添加混淆

  • 加载静态资源，也即是本身ReactNative目录下的图片(非Android或者iOS目录)下的本地照片。我们可以指定大小，也可以不指定(这时候显示图片原本大小)

  • 加载网络图片
    必须指定高度和宽度，否则无法显示出来。

  • 加载原生图片，比如Android目录下的资源的drawable文件以及ios目录下的对应的文件。
    需要注意新加图片假如是加到原生目录下面，需要重新运行才可以。

  • 加载手机本机图片

  • 使用Image组件时候的需要注意的问题

  • Image常见的属性
    1 style 属性，它包括
    a. Layout Props
    b. Shadow Props
    c. Transforms
    d.borderRadius 边框半径，还可以设置独立的上下左右四种不同的边框半径
    e. borderColor
    f. opacity:透明度，1不透明，0完全透明
    g. overflow：
    h. tintColor 给图片着色
    i. overlayColor

2 resizeMode
图片的缩放模式，默认cover，就是保持图片宽高比，且都大于等于容器的宽和高。contain:保持图片宽和高的前提下缩放图片，知道宽和高都小于容器的宽和高。center:居中不拉伸，repeat:重复平铺知道填满容器，仅ios可用。stretch:拉伸且不维持图片宽高比，知道宽和高都填满容器。

3 相关回调
a. onLoad 加载完成时候调用
b. onLoadStart 图片开始加载时候调用
c. onError 加载失败时候调用
d. onLoadEnd 加载结束调用
e. 对于iOS还有一个onProgress回调。

4 defaultSource 占位图，iOS才有效，他的使用跟source一样。

5 source 资源位置，本地照片使用require(path)，远程照片使用uri:xxx.jpg

Touchable系列组件

Touchable系列组件包括:

  1. TouchableWithouFeedback 响应用户事件，没有任何视觉反馈
  2. TouchableHighlight 在TouchableWithouFeedback基础上添加，当按下时，包裹视图的不透明度降低，使视图变暗或者着色。
    3.TouchableOpacity 他会降低背景透明度，但是不会改变背景颜色
  3. TouchableNativeFeedback 这是Android平台的水波纹效果。
    对于Touchable系列组件，他最多只能包裹一个子组件，去设置onPress，onLongPress等方法。

TouchableWithouFeedback使用

属性：

  • delayLongPress 延迟多少毫秒响应onLongPress事件
  • delayPressIn 延迟多少ms，之后响应onPressIn
  • delayPressOut 延迟多少ms，之后响应onPressOut
  • disabled是否激活可用
  • onLongPress 长按
  • onPress 点击
  • onPressIn 按下时的事件
  • onPressOut 触摸提起时的事件

    TouchableHighlight

属性：
它具有TouchableWithoutFeedback的所有属性，另外还有部分自己的属性

  • activityOpacity:激活的透明度
  • underlayColor 按下时候的颜色
  • onHideUnderlay() 底色不见的时候回调，也就是手机抬起的时候
  • onShowUnderlay 显示底色的时候回调，就是手指按下的时候。

TouchableOpacity

他只是具有一个透明度的变化，没有一个背景颜色的改变

他具有TouchableWithouFeedback的所有属性，并且还有一个属性：

  • activityOpacity 透明度，1位不透明，0全透明
